    • Клиент (DApp), желающий приобрести услуги хранения по установленным тарифам
    • Оплата тарифа по фиксированным в $ расценкам
    • Покупка CST для клиента по рыночной цене на бирже для внесения оплаты за услугу хранения через смарт-контракт
    • Фонд для поддержки функционирования работы сервиса, службы работы с клиентами и развития проекта
    • Смарт-контракт выполняет все операции по распределению данных по поставщикам и обрабатывает информацию о хранимых файлах от поставщиков
    • Формируется общий фонд вознаграждения в CST для всех поставщиков
    • Поставщик получает вознаграждение в CST от смарт-контракта в соответствии с вкладом в систему (выставленным/реальным объемом хранения)
